CSS (cascading style sheet)
스타일 지정을 위해서 사용하며 최신 버전은 3

```
<head> 주로 태그 안에 들어감.
<style> 이라는 태그에 바로 작성하기도 함.
선택자가 들어옴.
선택자 - 대표적으로 태그
{스타일 속성 : 속성값;}
</style>
</head>
css 기본 형식
선택자{스타일 속성 : 속성 값; 스타일 속성 : 속성값; ….}
```

### 1.basic.html

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <style>
      @import url('https://fonts.googleapis.com/css2?family=Do+Hyeon&display=swap');
      * {
        font-family: 'Do Hyeon', sans-serif;
      }
      h2 {
        color: red;
      }
      p {
        color: blue;
        font-size: 40px;
      }
    </style>
    <title>Document</title>
  </head>
  <body></body>
</html>
```

- 스타일 속성을 지정할 때 여러 개를 지정할 경우 줄바꿈을 이용하여 지정해주는 것이 가독성을 더 높여줄 수 있음
- html 코드 안에 CSS를 적용시켜주는 기법을 내부 스타일 시트라고 부름

### 2.outcss.html

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link rel="stylesheet" href="style.css" />
    <title>Document</title>
  </head>
  <body>
    <h1>흠흠 글자가 이쁘니까 맘에 드는군</h1>
    <h2>
      &lt;link rel="stylesheet" href="style.css"&gt; <= 이거 화면에 보이게 하는거 어케하게요??~
    </h2>
    <h3>흠흠</h3>
    <p>흐흐흐흠</p>
  </body>
</html>
```

- 외부에서 .css 파일을 불러오는 기법을 외부스타일 시트라고 부름
- css파일이 준비되어 있어야 하고 link 태그를 통해 불러옴
- link 태그를 사용하여 파일을 지정하고 css 파일임을 명시해주는 rel 속성

### 3.inline.html

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <link rel="stylesheet" href="style.css" />

  </head>
  <body>

```

- 해당 줄을 특정지어서 스타일을 지정하는 기법을 인라인 스타일시트라고 부름
- 다른 태그들과 다른 줄들은 해당 스타일의 영향을 받지 않

### 5_tag.html

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        @import url('https://fonts.googleapis.com/css2?family=Do+Hyeon&display=swap');

        h2{
            color: red;
        }
    </style>
    <title>Document</title>
</head>
<body>
    <h2?>헤헤</h2>
</body>
</html>
```

- 태그를 직접 지정하여 해당 모든 태그에 동일한 스타일을 지정할 수 있음

### 6_class.htm

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <style>
      @import url('https://fonts.googleapis.com/css2?family=Do+Hyeon&display=swap');

      .red {
        color: red;
      }
    </style>
    <title>Document</title>
  </head>
  <body>
    <h2 class="red">헤헤</h2>
  </body>
</html>
```

- 클래스명을 이용하여 클래스 선택자 생성이 가능하며 필요한 부분에 class=”클래스명” 을 지정하여 스타일 지정 가능

### 7_id.html

```html
<!doctype html>
<html lang="en">
```

- id 속성은 html 태그 내에서 id 속성을 사용할 경우 겸사겸사 스타일 지정할 때 사용
- 클래스는 css에서 보편적으로 사용하는 기술이고 id는 클래스보다 희소성을 갖고 사용을 함

클래스와 id 차이점

- 클래스는 보편적으로 사용
- id는 html id 속성과 사용함 input, label 등

### 8_cascading.htm

- 규모순 : 인라인 < id 선택자 < class 선택자 < 태그 선택자 < 전체 선택자
- 우선순위 : 인라인 > id 선택자 > class 선택자 > 태그 선택자 > 전체 선택자

### 9_important.htm

- 우선순위를 무시하고 최상의 우선순위를 갖기 위해서는 !important를 문주에 넣어주면 됨
- !important 로 지정한 스타일 형식이 두 개 이상이라면 내부적으로 우선 순위를 따져봄

### 10.font-family

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <style>
      @import url('https://fonts.googleapis.com/css2?family=Do+Hyeon&display=swap');
      * {
        font-family: 'Do Hyeon', '맑은 고딕', 돋움, 굴림;
        color: red;
      }
    </style>
    <title>Document</title>
  </head>
  <body>
    <h2 id="red">헤헤</h2>
  </body>
</html>
```

- font를 지정하기 위해서는 font-family를 사용함
- 폰트는 여러가지 지정이 가능하며 글꼴을 순차적으로 적용시킴(없으면 다음 글꼴을 찾아서 적용)
- 웹을 제공받는 사용자도 폰트를 가지고 있어야 하는 특징이 존재
- 맑은 고딕에는 띄어쓰기가 있기 때문에 묶어줌(단일 문자열은 따옴표를 필요로 하진 않지만 묶어줘도 상관 없음)
- 구글에서 제공해주는 웹 폰트를 이용하여 사용자들에게 글꼴 제공 가능
- 웹을 제공해주는 서버 측이나 클라이언트 측에서는 글꼴이 필요하지 않음
- 웹 폰트로 제공 되는 글꼴만 사용 가능

### 11.로컬 폰트

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <style>
      @font-face {
        font-family: 'trana';
        src:
          local('trana'),
          url('trana.eot') url('trana.woff') format('woff'),
          url('trana.ttf') format('truetype');
      }
      .t-font {
        font-family: trana, 'Noto Serif KR', 'Do Hyeon', sans-serif;
        color: red;
      }
    </style>
    <title>Document</title>
  </head>
  <body>
    <h2 class="t-font">으아아아아아가나다라마바사아아아아아아자차카타파하하하하하</h2>
    <h2 class="t-font">
      font test, if i test this font, you will be regret or die after clean your room
    </h2>
  </body>
</html>
```

- 폰트를 웹에 직접 적용하면 클라이언트 측에 글꼴이 없더라도 적용 가능
- eot(구버전 브라우저에서 사용하는 확장자), woff(웹 전용 확장자), ttf(글꼴 자체 확장자) 등을 사용하여 등록 가능

### 16.list_style_type

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <style>
      .p {
        list-style-type: none;
      }
      .c {
        list-style-type: circle;
      }
    </style>
    <title>Document</title>
  </head>
  <body>
    <ul class="p">
      <li>1</li>
      <li>2</li>
      <li>3</li>
    </ul>
    <ul>
      <li class="c">4</li>
      <li>5</li>
      <li>6</li>
    </ul>
  </body>
</html>
```

- list-style-type 속성을 이용하여 불릿 스타일 적용 가능
- none(불릿 없애기), circle(빈 원), disc(채운 원), square(채운 사각형) 값 사용 가능
- 상속의 개념으로 상위(ul)에 스타일을 적용해주면 하위(li)에도 전부 적용이 됨 → 태그 상속
