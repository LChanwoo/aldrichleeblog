### MAC Address

- 하드웨어 주소
- only + Unique
- 48자리의 이진수

### IP Address

인터넷 세상에서 찾아가기 위한 논리적인 주소

- 라우터를 통한 다른 네트워크와 통신하는 넓은 범위의 통신을 위해서 필요
- TCP/IP의 심장부로서 인터넷층의 프로토콜
- 이진수 32자리 → 8자리마다 (옥테트)점으로 구분 → 실제사용은 십진수

|  | 공인 IP | 사설 IP |
| --- | --- | --- |
| 할당주체 | ISP(인터넷 서비스 공급자 | 라우터(공유기) |
| 할당대상 | 개인 또는 회사의 서버(라우터) | 개인 또는 회사의 기기 |
| 고유성 | 인터넷 상에서 유일한 주소 | 하나의 네트워크 안에서 유일 - 네트워크가 다를경우 중복 가능 |
| 공개여부 | 내/외부 접근가능 | 외부 접근 불가능 |
|  |  | 사설 IP = 가상 IP/ 로컬 IP |

| MAC | IP |
| --- | --- |
| local Identification | global identification |
| layer2 operation | layer3 operation |
| physical address | local address |
| 48bits | 32bits |

### IP 주소 CLASS

- 5개로 구분
- 네트워크의 크기에 따라서 구분
- (= 하나의 네트워크가 호스트를 몇개 까지 가질 수 있는가)
- 궁극적으로 IP주소를 적절하게 효율적으로 배분하기 위한 목적

### IP주소 CLASS A

- 하나의 네트워크가 가질 수 있는 호스트가 가장 많은 수
- 대규모 네트워크
- 32개의 이진수 중에서 맨 앞쪽 하나가 항상 0으로 시작(나머지는 0 과 1 중에 아무거나 나와도 상관없음)
- 0.0.0.0 ~ 127.255.255.255
- 실제사용 IP 1.0.0.0 ~126.0.0.0
- 클래스 A의 호스트 수 : 2의 24승 - 2 = 16,777,214
    
    (모두 0인 경우는 네트워크 자체 // 모두 1인 경우는 브로드캐스트 주소)
    
- 0.0.0.0 제외 / 127.0.0.0 제외 (약속된 부분)

### IP주소 CLASS B

- 중규모 네트워크
- 128.0.0.0 ~ 191.255.255.255
- 클래스B 의 호스트 개수 = 2의 16승 -2 = 65,534
- 맨 앞이 반드시 10(이진수)로 시작(나머지는 0과 1중에서 아무거나 나와도 상관없음)
- network.network.host.host

### IP주소 CLASS C

- 소규모 네트워크
- 맨 앞이 반드시 110(이진수)로 시작
- 192.0.0.0 ~ 223.255.255.255
- 클래스 C의 호스트 수 : 2의 8승 - 2 = 254개
- network.network.network.host

### 네트워크 vs 호스트

- 호스트 : 내 위치(PC)
- 네트워크 : 내 도메인
